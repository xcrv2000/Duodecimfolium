## 文档维护规范

### 更新流程

1. 修改设计或代码时，**同时更新相关文档**
2. 在文档顶部更新"最后更新"日期
3. 在 `/reference/logs/CHANGELOG_*.md` 中记录变更
4. 若为破坏性变更，在迁移指南中补充

### 文档模板

设计文档应包含：
- **标题和日期**
- **概述**（核心设计思想）
- **详细说明**（具体实现细节）
- **示例**（代码或流程图）
- **相关链接**（指向源代码或其他文档）

### 版本管理

- v0.2.0 及以后版本的文档变更记录在 `/reference/logs/` 中
- 各版本对应的文档快照可从 git history 中查询
- 最新设计文档位置：`/reference/` （准备迁移至 `/reference/docs/`）

## 贡献指南

若要补充或修改文档：

1. **编辑文件** - 在 `/reference/docs/` 中编辑
2. **更新此索引** - 在"文档导航"部分加入新文档
3. **记录变更** - 在 `/reference/logs/CHANGELOG.md` 中记录
4. **提交审查** - 确保文档与代码保持一致
