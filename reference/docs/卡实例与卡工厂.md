# 卡实例与卡工厂机制 v0.4.0

## 文档定位

- `reference/docs/战斗系统修正文档.md` 中保留了旧版流程描述（含已过时段落），用于历史追踪。
- 如与旧版描述冲突，以当前代码与本文档为准。

## 概述

战斗系统中存在两个核心概念：**卡工厂（CardFactory）** 和 **卡实例（CardInstance）**。

### 卡工厂 (CardFactory)
- 存储在 `src/data/cards.json` 中，是卡的配置模板
- 包含卡的基础属性：名字、生效刻、脚本ID、标签等
- 在游戏启动时加载，不会改变
- 每次卡执行时使用的脚本由卡工厂的 `scriptId` 决定

### 卡实例 (CardInstance)
- 运行时的卡对象，在战斗初始化时为每张卡创建一个实例
- 包含卡的动态属性：当前生效刻、buff、持有者ID等
- 存储在 `BattleUnit.cards[]` 数组中
- 卡实例执行后会留在时间轴上，下一个时刻如果没有其他卡可执行，它仍可能被再次执行

## 时间轴系统

### 卡的执行流程
1. **战斗初始化** → 为每个战斗角色的卡组生成卡实例
2. **每一tick（刻）** → BattleLoop 找出生效刻等于当前tick的卡实例，执行其脚本
3. **卡实例重用** → 同一张卡实例可能在多个tick内执行多次（只要其所有者还活着，且卡未被删除）
4. **角色死亡** → 该角色的所有卡实例被从时间轴上删除

### 关键特性
- 卡实例不会在回合结束时自动删除
- 卡实例会在以下情况被删除：
  - 角色死亡
  - 被特殊卡牌效果删除（如【气定神闲】、【休眠】等）
  - 使用"离场召唤物"机制（如猎鹰）

## 休眠（Hibernate）机制 v0.4.0

### 效果说明
- 执行时：删除自己的卡实例 + 删除目标卡的卡实例
- 删除 **卡工厂** 是指删除自己这张【休眠】卡的卡实例，使其不再出现在时间轴上

### 为什么同时删除两张卡？
- **删除目标卡实例**：使目标卡在当前回合停止执行
- **删除自己的实例**：【休眠】卡本身也执行了，需要从时间轴上移除，以免下一个tick再次执行

### 实现逻辑
1. 找出时间轴上的下一张卡（根据生效刻）
2. 从持有者的 `cards[]` 数组中删除下一张卡
3. 从 `cards[]` 数组中删除自己的卡实例
4. 不涉及工厂定义的修改（工厂定义是只读的配置）

### 结果
- 第一回合：目标卡被删除，休眠卡被删除 → 都不会再执行
- 第二回合及以后：因为两张卡的实例都已被删除，它们都不会出现

这等效于"重新生成时跳过这两张卡"的效果。

## 多人战斗中的卡实例

### 多个角色的卡实例进入同一时间轴
- 所有角色的卡实例按生效刻排序进入同一个时间轴
- 执行顺序由生效刻决定，不由角色顺序决定

### 角色死亡时的处理
- 角色死亡时，其所有卡实例被立即从时间轴上卸载
- 即使有技能能复活角色，卸载的卡实例也不会自动加回

## 衍生卡的运行时生成

某些卡（如【猎犬口哨】）可能在执行时为其他角色动态添加卡实例。这些卡实例：
- 由 `BattleLoop.addTokenCardToUnit()` 添加
- 拥有相同的持有者ID（持有者是被添加卡的角色）
- 在时间轴的末尾被加入（生效刻会在所有现有卡之后）

## v0.4.0 相关API

### 核心卡实例操作方法
- `BattleLoop.executeCard(source, card)` → 执行卡实例
- `BattleLoop.addTokenCardToUnit(target, cardId, speed)` → 为角色添加衍生卡
- `BattleLoop.findNextCardOnTimeline(unit)` → 找出即将执行的下一张卡
- `BattleLoop.modifyCardSpeed(card, delta10)` → 修改卡的生效刻

### 工厂级别的操作（不常用）
- 大多数操作在卡实例级别，不涉及工厂定义修改
- 工厂定义来自 `cards.json` 且仅在启动时加载

## 设计理念

这套系统的设计目的是：
1. **灵活性** → 同一张卡可能执行多次，支持多次触发的卡牌设计
2. **可控性** → 可以精确删除特定卡实例，实现"跳过"、"禁用"等效果
3. **多人支持** → 多个角色的卡进入同一时间轴，完全基于生效刻排序

## 示例：标准的四角色战斗

### 初始化阶段
```
角色 A (玩家角色1) → 卡组 [卡A1, 卡A2, 卡A3]
角色 B (玩家角色2) → 卡组 [卡B1, 卡B2]
敌人 E1 → 卡组 [卡E1a, 卡E1b]
敌人 E2 → 卡组 [卡E2a]

合并时间轴（按生效刻排序）:
Tick 0: 卡E2a (速度0)
Tick 2: 卡A1 (速度2)
Tick 3: 卡B1 (速度3)
Tick 4: 卡E1a (速度4)
...
```

### 执行过程中（假设执行了三个tick）
```
Tick 0: 执行卡E2a → 卡E2a 仍在 unit.cards 中
Tick 2: 执行卡A1 → 卡A1 仍在 unit.cards 中
Tick 3: 执行卡B1 → 卡B1 仍在 unit.cards 中
```

### 如果角色A的卡A1被【休眠】删除
```
原时间轴: [卡E2a, 卡A1, 卡B1, 卡E1a, ...]
删除卡A1后: [卡E2a, 卡B1, 卡E1a, ...]

→ 卡A1 再也不会执行
→ 卡A2 和卡A3 可能仍在时间轴上执行
```
